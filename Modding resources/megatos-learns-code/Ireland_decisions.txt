# ** Notes **
# Using variables and testing
# In HOI4 variables can only store numbers, NOT strings / char. 
# A variable with no value assigned is by default 0.
# Make sure to declare the variable in correct scope for intended use. 
# Or else if the variable dont exist it will cause errors upon attempting to access it.
# Localisation: Variables and flags wont display unless localisation is made for them.
# Testing needs to be done through decisions/console commands.
# Temporary variables are useful to reduce impact on performance.
#	Values stored inside temp.vars are deleted once the code is executed. 
# Outside of HOI4 Example of temp.var: variable defined in a method, instance-variables for objects. 
# Temp.var can normally be stored forever by declaring them static. 
#####################################################################
# When to use variables?
# Working with values that needs to be changed, or generated in-game.
# Variables are useful for counting (just as in any other program)
# Variables can be used in focuses/decisions/events effect and / or conditions
#####################################################################
# Basic variable commands:
# set_variable - creates variable and sets it's value to the given value
# add_to_variable - += to variable
# substract_from_variable - -= to variable
# multiply_variable - *= to variable
# clear_variable - clears the variable, deleting it from the game. C: free(var), python: del var
# round_variable - rounds off to closest integer
# (trigger) check_variable - checks the value of a variable 
# effect = var:varName - Uses the variable to modify the effect with its value
# var:TAG.varName - TAG. Looks at scope to other countries
